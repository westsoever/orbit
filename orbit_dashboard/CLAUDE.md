Build a single-file dashboard.html that visualizes a littlebird daily report.

REFERENCE FILE: chieff_of_staff_v1.md (a sample report — use it for parser
testing and field-mapping verification).

REQUIREMENTS

Stack:
- Single dashboard.html, no build step, opens locally
- Tailwind CSS via Play CDN
- marked.js via CDN for markdown rendering
- All state in localStorage (work hours, theme, last report)

Aesthetic:
- Catalyst (Tailwind UI) inspired — zinc neutral palette, rounded-xl cards,
  subtle borders, generous whitespace, system font stack
- Dark + light mode with toggle button, persisted
- Production-grade visual quality, not generic AI dashboard look
- Smooth transitions, restrained animations

Layout (top to bottom):
1. Top bar: live clock (updates every 1s) + "Xh Ym left at work" + settings
   gear (opens modal with start/end time inputs) + theme toggle
2. Hero row (3 cards): productivity score gauge | top priority | daily quote
3. Calendar timeline (vertical, time-chipped list, today's events from report)
4. Day Insights — single accordion with 3 unfoldable sub-cards:
   Recommendations for tomorrow | What to leverage | This week's todos
5. Full report sections (collapsed by default): Communications Triage,
   Action & Insight Ledger, Daily Growth Reflection, Key Moment Analysis
6. Bottom: drag-and-drop zone for loading a new .md report

Parser:
- Split markdown on `^### \d+\.\s+` to get numbered sections
- Section 2 ("Tomorrow's Horizon"): extract "Critical Priority" paragraph and
  parse "Scheduled Engagements" — each item matches `\*\*(\d{2}:\d{2})\s*–\s*(\d{2}:\d{2}):\*\*\s*(.+)`
- Section 6 ("Productivity Score"): extract score via `Score:\s*([\d.]+)\s*\/\s*10`,
  extract "Recommendations for Tomorrow" sub-list
- Section 4 ("Daily Growth Reflection"): extract "Success Observation" for
  Leverage card
- Section 3 ("Action & Insight Ledger"): extract "New Commitments & Follow-ups"
  list for weekly todos
- Everything else: pass raw markdown into marked.parse() inside its card
- Graceful degradation: missing fields → hide card, never crash

Score color logic:
- 0–4.9 red-500, 5–6.9 amber-500, 7–8.4 lime-500, 8.5–10 emerald-500
- Apply to gauge fill and the score number

Calendar highlight:
- The event whose title matches the Critical Priority gets a ring-2 ring-amber-400
  accent

Work hours config:
- Settings modal with two <input type="time"> fields
- Defaults: 09:00–17:00
- "Hours left" = max(0, end - now) when now is within window; "Workday done"
  outside it

Polish:
- Loading state when parsing
- Clear error toast if file isn't a valid littlebird report (missing §6 score)
- Keyboard shortcut: `t` toggles theme, `,` opens settings
- Smooth accordion transitions (max-height + transform)

Build it, then test by loading chieff_of_staff_v1.md and verify all fields
render correctly.
